What is Elegant Agent MCP?

An MCP server that converts prompts into fully functional HTML screens for Log360 Cloud.

Web Components (CDN)

All UI atoms (<me-*> tags) are served from cloud CDN. No build step needed.

3 Tools Only

plan_feature(), get_atoms(), modify_screen() — simple and composable.

15 Shell Layouts

Pre-designed layout skeletons (A–O) for every screen type.

How It Works

You describe a feature in natural language. The AI automatically:

1 Plans the feature
→
2 Selects shell layout
→
3 Fetches CDN atoms
→
4 Writes HTML file
→
5 Open in browser
Internet Required

Generated HTML loads web components from a cloud CDN. You must be online to view pages in your browser.

Scenario 3: Recreating from a Screenshot (Area Map)

When you have a screenshot of an existing UI and want to recreate it, you must provide context using the Area Map approach.

Area Map = describe what you see in the screenshot

Describe the layout zones from left→right, top→bottom. This helps the AI pick the right shell and recreate it accurately.

Screenshot Recreation Prompts

// Attach your screenshot + describe it "Recreate this screen for Log360 Cloud — use Elegant MCP tool Area Map: - Top: Navigation bar with Dashboard tab active - Row 1: 4 KPI metric cards (Total Events, Alerts, Sources, Latency) - Row 2 Left: Area chart showing event trend over 7 days - Row 2 Right: Donut chart showing event categories - Row 3: Full-width data table with recent events Product: Log360 Cloud Tab: Dashboard"
// Sidebar layout with report table "Recreate this screen for Log360 Cloud — use Elegant MCP tool Area Map: - Left: Side menu with report categories (Logon Reports, Account Reports) - Top Right: Filter row with source picker and date range - Middle Right: Bar chart showing logon activity by hour - Bottom Right: Data table with columns (User, Source, Time, Status) Product: Log360 Cloud Tab: Reports"
// Settings page recreation "Recreate this screen for Log360 Cloud — use Elegant MCP tool Area Map: - Left: Settings side menu (Notifications, Profiles, Policies) - Right Top: Classic tabs (Email, SMS, Webhook) - Right Body: Form fields for SMTP config (server, port, encryption) - Right Bottom: Save/Cancel buttons Product: Log360 Cloud Tab: Settings"

Area Map Best Practices

Describe Top → Bottom

List zones in reading order: top bar, then row by row, left to right.

Name the Components

Say "KPI cards", "data table", "donut chart", "side menu" — specific names help.

Mention Product & Tab

Always end with "Product: Log360 Cloud" and "Tab: [name]" for proper context.

Say "use Elegant MCP tool"

This triggers the AI to use plan_feature() with has_image=true for screenshot matching.

Scenario 4: Multiple Layout Options (Shell X — AI Custom)

When you want the AI to design a completely new layout from scratch — or generate multiple design options for the same page using competitor patterns — use Shell X.

Shell X = AI designs the layout from zero

Unlike shells A–O which are pre-built skeletons, Shell X tells the AI to create a custom layout. Combine with competitor patterns to get multiple unique design options.

How It Works

Attach a screenshot → describe the Area Map → ask for multiple layout variations → AI generates different designs using competitor patterns and assigns Shell X.

// Attach screenshot + ask for design options "Here's my current dashboard. Create an Area Map from this screenshot. I want 3 different layout designs for the same page using competitor patterns. Use Shell X. Product: Log360 Cloud Tab: Dashboard"
// Ask AI to research competitors and give layout options "Recreate this screen for Log360 Cloud — use Elegant MCP tool Area Map: - Top: KPI cards row - Middle: 2-column chart grid - Bottom: Alert feed table Give me 3 design variations inspired by: 1. Splunk SIEM layout 2. Microsoft Sentinel style 3. CrowdStrike dashboard pattern Use Shell X for custom layout. Product: Log360 Cloud"
// No screenshot — pure AI creativity with Shell X "Design a Threat Hunting Dashboard for Log360 Cloud using Shell X — I want a unique layout not based on any predefined shell. Research top SIEM competitors and create something modern with real-time data emphasis."
// After AI gives you multiple layout options, pick one "I like Option 2 (the Sentinel-inspired layout). Build that as the final HTML — use Shell X."

When to Use Shell X

Want Multiple Options

Ask for 2–4 layout variations and pick the one you like best before building.

Competitor Research

Reference specific products (Splunk, Sentinel, CrowdStrike) for layout inspiration.

No Shell Fits

When your page doesn't match any predefined shell (A–O), Shell X builds custom.

Creative Freedom

Let the AI design an entirely new layout structure — no constraints from templates.

The 3-Step Workflow

How the AI builds your screen behind the scenes.

Step 1: plan_feature()

Triggered automatically when you say "create", "build", "design", or mention a feature name.

// You type: "Create Event Overview Dashboard for Log360 Cloud" // AI calls internally: plan_feature( feature_name = "Event Overview Dashboard", description = "Dashboard showing event metrics, trends..." ) // Returns: feature_plan_id + shell_guide table

Step 2: get_atoms()

AI picks the best shell and fetches all web component templates + layout rules.

get_atoms( feature_plan_id = "abc123", shell = "A", // Dashboard shell filename = "event-overview-dashboard.html" ) // Returns: 31 CDN atoms + shell_contract + scaffold + CSS tokens

Step 3: Write HTML

AI assembles the page using <me-*> web component tags and writes the file.

Zero Code from You

You don't write any code! Just describe what you want, and the AI handles the entire pipeline.

Architecture & Cloud CDN

Understanding how the generated HTML connects to cloud-hosted web components.

Internet Required to View HTML

All UI components (<me-*> web components) are loaded from a Cloud CDN. You must have internet access to view generated pages.

System Architecture

┌─────────────────────────────────────────────────────────────┐ │ 🖥️ YOUR BROWSER │ │ ┌────────────────────────────────────────────────────────┐ │ │ │ event-overview-dashboard.html (local file) │ │ │ │ │ │ │ │ <me-widget> <me-kpi-strip> <me-data-table> │ │ │ │ ↓ ↓ ↓ │ │ │ └───────┼──────────────┼───────────────┼─────────────────┘ │ └──────────┼──────────────┼───────────────┼───────────────────┘ ↓ ↓ ↓ ┌─────────────────────────────────────────────────────────────┐ │ ☁️ CLOUD CDN │ │ ├── tokens.css (design tokens — colors, spacing) │ │ ├── atoms.css (component styles — 31 atoms) │ │ ├── layout.css (shell layout system) │ │ ├── atoms.js (web component logic) │ │ ├── ECharts (charting library) │ │ └── Lucide Icons (638 icons) │ └─────────────────────────────────────────────────────────────┘

Local vs Cloud

Stored Locally

HTML files, page-specific CSS, content data — all in your workspace folder.

Loaded from CDN

tokens.css, atoms.css, atoms.js, layout.css, ECharts, Lucide icons.

For users on Claude (not Cursor IDE) — how to install the MCP and configure your project.

Install MCP (One-Time Setup)

Claude Desktop can install and connect to the Elegant Agent MCP server automatically. Just paste the link and let Claude handle it.

Step 1: Open Claude Desktop

Launch Claude Desktop app on your machine. Make sure you're on the latest version.

Step 2: Go to AI Code Mode

Switch to the AI code mode in Claude Desktop (where it can execute commands).

Paste This Prompt in Claude

"Use this MCP tool link and make it work. Install all prerequisites if needed: https://elegant-public-1042350023.development.localcatalystappsail.com/"
Claude handles the rest automatically

Claude will detect it's an MCP server URL, install any required dependencies (Node.js, npm packages, etc.), configure the connection, and confirm when it's ready.

Verify Installation

Step 3: Restart Claude Desktop

After installation completes, close Claude Desktop completely and reopen it. This is required for the MCP to register.

Step 4: Confirm in Settings

Go to Settings → Developer. You should see "Elegant Agent" listed as a running MCP connector.

You'll see "Elegant Agent" in the connector list

If it shows as connected/running in Settings → Developer, you're all set! The MCP is now active for all conversations.
